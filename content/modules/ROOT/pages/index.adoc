= AI-Powered Database Analytics with Built-in Governance

Welcome to the AI-Powered Database Analytics with Built-in Governance workshop!

== What you'll learn

In this workshop, you will:

* Understand the basic definition and building blocks of an agentic AI application, including concepts like the agentic loop, Model Context Protocol (MCP), and how agentic AI differs from other forms of AI
* Learn the design decisions that must be made when building an agentic AI application
* Explore the architectural elements that comprise the Data Governance Copilot quickstart
* Evaluate the choice between custom orchestration and third-party frameworks like Llama Stack
* Understand the specific use case and functionality the Data Governance Copilot quickstart addresses
* Discover the benefits of the Data Governance quickstart, including leveraging an enterprise's investment in relational data and achieving data sovereignty using OpenShift AI

== Who this is for

This workshop is designed for developers, architects, SREs, and technical/business decision makers who want to understand how to build AI-powered analytics with built-in governance controls.

**Experience level**: Beginner to Intermediate

== Prerequisites

Before starting this workshop, you should have:

* Basic understanding of databases and SQL
* Familiarity with AI/ML concepts (helpful but not required)
* Basic knowledge of OpenShift (helpful but not required)

== Workshop environment

You will work with a pre-deployed Data Governance Copilot environment running on Red Hat OpenShift. The environment includes:

* OpenShift AI 3 with inference capabilities
* EnterpriseDB's pg-airman-mcp server for database introspection
* PostgreSQL 15+ with sample e-commerce data
* A web-based copilot interface for conversational analytics
