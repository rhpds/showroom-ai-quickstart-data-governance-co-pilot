== Technical requirements

=== Software versions
* Red Hat OpenShift Container Platform 4.20 or later
* Red Hat OpenShift AI 3.0 or later
* PostgreSQL 15 or later
* Web browser (Chrome, Firefox, Safari, Edge)

=== Environment access
Participants need access to:

* OpenShift AI cluster (provided during workshop)
* Data Governance Copilot web UI
* OpenShift web console
* Terminal/command line access via web terminal

=== Network requirements
* Internet connectivity for accessing the OpenShift cluster
* Access to OpenShift routes and services
* HTTPS access to the copilot web interface

== Workshop components

=== Pre-deployed infrastructure
The following components are already deployed and configured:

* **OpenShift AI 3**: Inference platform with vLLM runtime
* **LLM Model**: Qwen3-14B or Nemotron model for natural language processing
* **pg-airman-mcp**: EnterpriseDB's Model Context Protocol server for PostgreSQL introspection
* **PostgreSQL 15+**: Database with sample TradeNexus e-commerce data including:
** Customer dimension table with state/region data
** Product catalog
** Transaction history
** Governance metadata and annotations
* **Copilot Backend**: FastAPI orchestration service (supports both MCP direct and Llama Stack modes)
* **Copilot UI**: Svelte-based conversational interface

=== Architecture overview
The Data Governance Copilot consists of these layers:

1. **User Interface**: Web-based chat for natural language queries
2. **Orchestration Layer**: Backend that implements the agentic loop
3. **MCP Server**: Database introspection and governance tools
4. **Inference**: OpenShift AI serving the LLM model
5. **Data Layer**: PostgreSQL with governance metadata
